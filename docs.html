Embed zaps on your website

This guide shows you how to add a Nostr zap button to your own website so visitors can send you zaps. Everything runs in the browser: the viewer reads zap receipts from relays, and the zap flow requests invoices from your profile‚Äôs zap endpoint.

Embed (plain HTML)

‚úÖ Step 0 ‚Äî Make sure you can receive zaps

Before you embed anything, your Nostr profile must be ‚Äúzap-ready‚Äù. If the zap button cannot find a Lightning address / LNURL in your profile metadata, it can‚Äôt generate an invoice.

What you need: a Lightning address (often stored as lud16) or an LNURL (often stored as lud06) in your Nostr profile metadata (kind 0). Most Nostr clients provide a field like ‚ÄúLightning address‚Äù or ‚ÄúZap address‚Äù.

üß† Mental model (quick)

  • Zap button ‚Üí looks up your profile, finds your zap endpoint, requests an invoice, then lets the user pay (WebLN or QR).
  • Viewer button ‚Üí reads zap receipts from relays and shows them in a dialog.

üëÄ Step 3 ‚Äî Add a ‚ÄúView zaps‚Äù button

The library auto-wires any button[data-nzv-id] on click and opens a dialog. Use it to show zaps for a profile, a note, or an addressable event.

Minimal viewer example

  data-relay-urls="wss://relay.damus.io,wss://relay.primal.net,wss://nos.lol"

  üëÄ View zaps

üß∑ Viewer attributes

  • data-nzv-id (required): Nostr identifier (npub, nprofile, note, nevent, naddr).
  • data-relay-urls (required): comma-separated relay URLs.
  • data-title (optional): dialog title (empty string is allowed).
  • data-zap-color-mode (optional): "true" or "false".
  • data-theme (optional): "light" or "dark". If missing, the dialog follows the user‚Äôs system theme.

üß∞ JavaScript API (browser global)

After you include the bundle using a <script> tag, the library is available as window.nostrZap. In most cases you don‚Äôt need to call anything manually adding the data-* attributes to your buttons is enough.

  • window.nostrZap.initialize() ‚Äî optional manual initialization
  • window.nostrZap.nostrZapView() ‚Äî open the zap viewer dialog (alias)
  • window.nostrZap.nostrZap ‚Äî legacy API object

üßæ Legacy zap buttons (still supported)

Older embeds remain compatible. The most common attributes are: data-npub, data-note-id, and data-relays.

// window.nostrZap.nostrZap.init({ npub, noteId?, naddr?, relays?, buttonColor?, anon? })

// Convenience helpers:

  npub: "npub1...",
  relays: "wss://relay.damus.io,wss://nos.lol",

window.nostrZap.zapInitTargets();

üì° Relays (recommended)

Your buttons should use multiple relays. For best results, use a mix of reliable read+write relays. The demo page ships with a curated list, but you can pass your own per button.

Zap flow: uses data-relays (string).
Viewer: uses data-relay-urls (comma-separated list).

üõ†Ô∏è Troubleshooting

  • ‚ÄúUnable to fetch invoice‚Äù: your profile likely has no Lightning address / LNURL in metadata, or the endpoint is unreachable.
  • No zaps show up: try different relays (zaps may not be replicated everywhere).
  • Theme not applied: set data-theme directly on the button that opens the dialog.
  • Payments: users with WebLN get a one-click flow; otherwise they can scan/copy the invoice QR.
